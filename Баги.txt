ВИПРАВЛЕННЯ (14.05.2026)
==========================

1. ПОМИЛКА: attendance.is_paid та attendance.payment_choice відсутні в БД PostgreSQL
   - SQLAlchemy модель має ці колонки, але в реальній БД на Railway їх немає
   - ФІКС: створено backend/migrate_attendance_columns.py — скрипт додає обидві колонки
   - Додано запуск міграції в railway.toml та start.bat

2. ПОМИЛКА: При редагуванні призначення payment_choice в учнів скидався
   - Причина: SQLAlchemy many-to-many (assignment_students) не передає extra колонки
   - ФІКС: backend/app/api/assignments.py — додано _attach_payment_choices() для GET/POST/PUT

3. ПОМИЛКА: Кнопка "Підтвердити" в відвідуваннях заблокована
   - Причина 1: payment_choice null → ставився 'subscription', але is_present: false (Pydantic default)
   - isPaid = false || (true && false) = false → всі неоплачені
   - ФІКС: frontend/js/attendance.js — змінено на isPaid = s.is_paid === true || choice === 'subscription'

4. ДОДАТКОВО: subscription decrement додано в PUT endpoint attendance
   - Раніше списання абонементу було тільки в POST (з upsert)
   - Тепер і PUT обробляє списання при зміні на 'subscription'

5. МІГРАЦІЯ В main.py: додано автоматичне створення колонок при старті
   - Тепер незалежно від способу запуску (railway.toml, Procfile, start.bat)
   - при старті додаються payment_choice та is_paid в таблицю attendance
   - Працює для PostgreSQL та SQLite
